0. Operazioni su di un Insieme

Operazione interna a un insieme

Un'operazione si dice interna a un insieme S se, prendendo due elementi di S, il risultato è ancora in S.
Esempio: la somma tra due vettori di ℝ² è ancora un vettore di ℝ².

Operazione esterna a un insieme

Un'operazione si dice esterna se coinvolge due insiemi diversi: uno su cui si agisce (S) e uno di operatori (A).
Esempio: la moltiplicazione tra un vettore di ℝ² e un numero reale: ℝ² × ℝ → ℝ².

Prodotto scalare standard

Il prodotto scalare tra due vettori di ℝⁿ restituisce un numero reale. Non è né interna né esterna. Si calcola così:
(a₁, ..., aₙ) · (b₁, ..., bₙ) = a₁b₁ + ... + aₙbₙ
  • Simmetria: a · b = b · a
  • Bilinearità: (h·a + k·b) · c = h(a·c) + k(b·c)
  • Positività: a · a ≥ 0 e a · a = 0 solo se a = (0, ..., 0)
Dimostrazione della bilinearità (passo passo):
1. Siano a = (a₁, a₂), b = (b₁, b₂), c = (c₁, c₂), h e k scalari.
2. Calcoliamo h·a + k·b:
  • h·a = (h a₁, h a₂) (moltiplichiamo ogni componente di a per h)
  • k·b = (k b₁, k b₂) (moltiplichiamo ogni componente di b per k)
  • h·a + k·b = (h a₁ + k b₁, h a₂ + k b₂) (sommiamo le componenti corrispondenti)
3. Facciamo il prodotto scalare con c:
(h a₁ + k b₁) c₁ + (h a₂ + k b₂) c₂ (prodotto scalare: moltiplichiamo ogni componente per quella corrispondente e sommiamo)
4. Sviluppiamo:
h a₁ c₁ + k b₁ c₁ + h a₂ c₂ + k b₂ c₂ (distribuiamo i prodotti)
5. Raggruppiamo:
h(a₁ c₁ + a₂ c₂) + k(b₁ c₁ + b₂ c₂) (raccogliamo h e k)
6. Riconosciamo che a·c = a₁ c₁ + a₂ c₂ e b·c = b₁ c₁ + b₂ c₂, quindi:
h(a·c) + k(b·c)
7. Abbiamo dimostrato che (h·a + k·b) · c = h(a·c) + k(b·c)
Ogni passaggio serve a mostrare come si applicano le proprietà delle operazioni tra vettori e scalari, e come si arriva alla forma finale.

1. Operazioni su Matrici (su ℝ)

Somma di matrici e prodotto per uno scalare

Una matrice è una tabella di numeri disposti su righe e colonne. Una matrice A con n righe e m colonne si scrive così:
a₁,₁...a₁,ₘ
.........
aₙ,₁...aₙ,ₘ
Oppure in modo compatto: A = (aᵢⱼ) con i = 1,...,n e j = 1,...,m.
L’insieme di tutte le matrici reali con n righe e m colonne si indica con Mₙ,ₘ(ℝ) o ℝₙ,ₘ.

Somma di matrici: si sommano elemento per elemento (solo se hanno la stessa dimensione):
a₁,₁ + b₁,₁...a₁,ₘ + b₁,ₘ
.........
aₙ,₁ + bₙ,₁...aₙ,ₘ + bₙ,ₘ
In notazione compatta: C = (aᵢⱼ + bᵢⱼ) con i = 1,...,n e j = 1,...,m.

Prodotto per uno scalare: ogni elemento della matrice viene moltiplicato per il numero scelto: h·A = (h·aᵢⱼ).

Prodotto righe per colonne

Il prodotto righe per colonne tra due matrici è possibile solo se il numero di colonne della prima matrice è uguale al numero di righe della seconda.
Se A è una matrice n × m e B è una matrice m × l, il prodotto AB è una matrice n × l (cioè ha il numero di righe della prima e il numero di colonne della seconda).

Per calcolare l’elemento nella posizione (i, j) del prodotto, si prende la riga i di A e la colonna j di B, si moltiplicano i termini corrispondenti e si sommano: 
(AB)ij = ai,1·b1,j + ai,2·b2,j + ... + ai,m·bm,j
In altre parole, ogni elemento del risultato si ottiene facendo il prodotto scalare tra una riga della prima matrice e una colonna della seconda.
Il risultato avrà tante righe quante la prima matrice e tante colonne quante la seconda matrice.

Proprietà del prodotto tra matrici:
  • Distributività: A(B + C) = AB + AC e (A + B)C = AC + BC
  • Non commutatività: in generale AB ≠ BA
  • Matrice identità: IₙA = AIₙ = A
  • Proprietà con scalari: A(hB) = h(AB) = (hA)B
  • Associatività: A(BC) = (AB)C
Dimostrazione dettagliata (passo passo):
1. Siano A di dimensione n × m e B di dimensione m × l.
2. Per poter moltiplicare A e B, il numero di colonne di A (m) deve essere uguale al numero di righe di B (m).
3. Il risultato C = AB sarà una matrice di dimensione n × l.
4. L'elemento cij si calcola così:
cij = ai,1·b1,j + ai,2·b2,j + ... + ai,m·bm,j
5. Si ripete questo calcolo per ogni coppia di riga (di A) e colonna (di B).
6. Alla fine, si ottiene una matrice in cui ogni elemento è il risultato di un prodotto riga per colonna.
Questo metodo permette di combinare le informazioni di due matrici in una nuova matrice, utile in moltissime applicazioni della matematica e della fisica.
Dimostrazione della distributività a destra (passo passo):
1. Siano A ∈ ℝₘ,ₙ e B, C ∈ ℝₙ,ₚ.
2. Consideriamo la somma B + C: sommiamo elemento per elemento.
3. Moltiplichiamo A per (B + C): ogni elemento della riga di A viene moltiplicato per la colonna corrispondente di (B + C).
4. Per la proprietà distributiva dei numeri reali, questa operazione equivale a fare A·B + A·C, cioè moltiplicare A per B e A per C separatamente e poi sommare i risultati.
5. Quindi A(B + C) = AB + AC.
Dimostrazione della proprietà con scalari (passo passo):
1. Moltiplichiamo la matrice B per lo scalare h: ogni elemento di B diventa h·bᵢⱼ.
2. Moltiplichiamo A per questa nuova matrice: il risultato è lo stesso che moltiplicare prima A per B e poi il risultato per h.
3. Oppure, possiamo moltiplicare prima A per h (cioè ogni elemento di A per h) e poi moltiplicare per B: il risultato è sempre lo stesso.
Dimostrazione dell'associatività (passo passo):
1. Siano A ∈ ℝₘ,ₙ, B ∈ ℝₙ,ₚ, C ∈ ℝₚ,ⱼ.
2. Calcoliamo prima BC: otteniamo una matrice.
3. Moltiplichiamo A per questa matrice: otteniamo il risultato finale.
4. Oppure, calcoliamo prima AB, poi moltiplichiamo il risultato per C.
5. In entrambi i casi, il risultato è lo stesso perché la somma degli indici intermedi è la stessa.

Matrice trasposta

Esempio:
Se A è la matrice
123
456
allora la sua trasposta è
14
25
36
In pratica, la prima riga diventa la prima colonna, la seconda riga diventa la seconda colonna, e così via.
La trasposta di una matrice si ottiene scambiando righe e colonne.
  • (AB)T = BTAT
  • (AT)T = A
  • (hA + kB)T = hAT + kBT
  • (A + B)T = AT + BT
  • (AT + BT)T = A + B
Dimostrazione di (AB)T = BTAT (passo passo):
1. Siano C = AB, D = (AB)T = CT, E = BTAT.
2. L'elemento dij di D è uguale all'elemento cji di CT (cioè scambiamo riga e colonna).
3. L'elemento cji di CT si ottiene facendo il prodotto riga per colonna tra la j-esima riga di A e la i-esima colonna di B.
4. Se calcoliamo BTAT, otteniamo lo stesso valore perché la trasposizione scambia l'ordine delle moltiplicazioni.
5. Quindi (AB)T = BTAT.

Matrice a gradini

Una matrice è a gradini se, andando verso il basso, il numero di zeri prima del primo elemento diverso da zero aumenta. Il primo elemento non nullo di ogni riga si chiama pivot.

Operazioni elementari di riga

Sono tre:
  1. Scambiare due righe
  2. Moltiplicare una riga per un numero diverso da zero
  3. Sostituire una riga con la somma di essa e un multiplo di un'altra riga
Queste operazioni permettono di trasformare una matrice in una matrice a gradini. Per ogni matrice esiste una matrice a gradini ad essa equivalente.
Dimostrazione (per induzione, passo passo):
1. Se la matrice ha una sola riga, è già a gradini.
2. Se ha più righe, si cerca la prima riga non nulla e la si porta in alto (scambio di righe).
3. Si annullano gli elementi sotto il primo elemento non nullo (pivot) usando le altre operazioni (moltiplicazione e somma di righe).
4. Si ripete il processo sulle righe restanti, riducendo il problema a una matrice più piccola.
5. Alla fine, si ottiene una matrice a gradini equivalente a quella di partenza.

2. Sistemi Lineari

Equazioni lineari

Un'equazione lineare è un'espressione del tipo a₁x₁ + ... + aₙxₙ = b, dove a₁, ..., aₙ e b sono numeri reali. Se b = 0, l'equazione si dice omogenea.

Sistema lineare

Un sistema lineare è un insieme di più equazioni lineari con più incognite. Una soluzione è un insieme di valori che soddisfa tutte le equazioni.
  • Un sistema è compatibile se ha almeno una soluzione, incompatibile se non ne ha.
  • Se ha una sola soluzione è determinato, se ne ha infinite è indeterminato.

Matrice dei coefficienti e matrice completa

La matrice dei coefficienti contiene solo i numeri davanti alle incognite. Se aggiungiamo la colonna dei termini noti otteniamo la matrice completa.

Sistemi equivalenti

Due sistemi sono equivalenti se hanno le stesse soluzioni. Le operazioni elementari sulle righe (scambio, moltiplicazione per un numero, somma di righe) non cambiano le soluzioni del sistema.
Dimostrazione (passo passo):
1. Scambiare due righe non cambia le soluzioni perché l'ordine delle equazioni non conta: ogni equazione rappresenta una condizione indipendente.
2. Moltiplicare una riga per un numero diverso da zero non cambia le soluzioni perché si può dividere entrambi i membri per quel numero, ottenendo la stessa equazione.
3. Sostituire una riga con la somma di essa e un multiplo di un'altra riga equivale a sommare due equazioni: le soluzioni che soddisfano entrambe le equazioni originali soddisfano anche la loro somma.
4. Quindi, tutte queste operazioni non cambiano l'insieme delle soluzioni.

Risoluzione dei sistemi

Ogni sistema può essere trasformato in uno a gradini (cioè con zeri sotto i pivot) usando le operazioni elementari. Questo rende più facile trovare le soluzioni.
Dimostrazione (passo passo):
1. Si applicano le operazioni elementari per portare la matrice in forma a gradini.
2. In questa forma, si parte dall'ultima equazione (che avrà meno incognite) e si risale, trovando una incognita alla volta (sostituzione all'indietro).
3. Questo metodo funziona perché ogni passaggio riduce il numero di incognite da determinare.

Soluzioni di un sistema lineare

A seconda del numero di equazioni (m) e di incognite (n), si distinguono diversi casi:
  • Caso 1: m = n = 1
    a₁₁x₁ = b₁
    • Se a₁₁ = b₁ = 0: equazione identica, infinite soluzioni.
    • Se a₁₁ ≠ 0: unica soluzione x₁ = b₁ / a₁₁.
    • Se a₁₁ = 0 e b₁ ≠ 0: nessuna soluzione.
  • Caso 2: m = 1, n > 1
    a₁₁x₁ + ... + a₁ₙxₙ = b₁
    • Se tutti i coefficienti e il termine noto sono zero: equazione identica, infinite soluzioni.
    • Se tutti i coefficienti sono zero ma il termine noto è diverso da zero: nessuna soluzione.
    • Se almeno un coefficiente è diverso da zero: si può scegliere liberamente n-1 incognite e calcolare la rimanente. Quindi ci sono infinite soluzioni (parametriche).
  • Caso 3: m, n > 1
    Si riduce il sistema a gradini e si eliminano eventuali equazioni identiche (0 = 0). Si ottengono i seguenti casi:
    • Se compare una riga del tipo 0 = c con c ≠ 0: il sistema è incompatibile (nessuna soluzione).
    • Se il numero di pivot (equazioni indipendenti) è uguale al numero di incognite (m = n): il sistema è determinato (unica soluzione).
    • Se il numero di pivot è minore del numero di incognite (m < n): il sistema è indeterminato (infinite soluzioni parametriche, alcune incognite sono libere).
Esempi:
  • Determinato: sistema con una sola soluzione.
  • Indeterminato: sistema con infinite soluzioni, alcune incognite sono parametri liberi.
  • Incompatibile: sistema senza soluzioni (ad esempio, una riga 0 = 2).
Nota: Un sistema omogeneo ha sempre almeno la soluzione banale (tutte le incognite nulle). Se ne ha una diversa, allora ne ha infinite.

3. Spazi Vettoriali

Definizione di Spazio Vettoriale

Sia V un insieme non vuoto, + : V × V → V un'operazione interna e · : ℝ × V → V un'operazione esterna. La terna (V, +, ·) è detta spazio vettoriale sul campo ℝ se:
  1. (V, +) è un gruppo abeliano, cioè gode di:
    • Proprietà associativa: (a + b) + c = a + (b + c)
    • Proprietà commutativa: a + b = b + a
    • Elemento neutro: ∃ε ∈ V tale che ∀a ∈ V, ε + a = a = a + ε
    • Opposto: ∀a ∈ V, ∃! b tale che a + b = ε
  2. ∀h, k ∈ ℝ, ∀v ∈ V risulta (hk) · v = h · (kv)
  3. ∀v ∈ V risulta 1 · v = v
  4. Distributività di · rispetto a + in ℝ: ∀h, k ∈ ℝ, ∀v ∈ V risulta (h + k)v = hv + kv
  5. Distributività di · rispetto a + in V: ∀h ∈ ℝ, ∀v, w ∈ V risulta h(v + w) = hv + hw
Gli elementi di V si chiamano vettori, quelli di ℝ scalari, + è detta addizione tra vettori, · è detta moltiplicazione di uno scalare per un vettore.

Proprietà fondamentali degli Spazi Vettoriali

Sia (V, +, ·) uno spazio vettoriale sul campo ℝ. Allora:
  1. Unicità dell'elemento neutro: ∃! elemento neutro rispetto a + (lo indichiamo con 0)
  2. Unicità dell'opposto: ∀v ∈ V, ∃! opposto per v (che indichiamo con -v)
Dimostrazione dell'unicità dell'elemento neutro:
Siano v₀ e w₀ elementi neutri. Allora v₀ = v₀ + w₀ = w₀
Dimostrazione dell'unicità dell'opposto:
Siano v' e v'' opposti di v. Abbiamo: v' = v' + 0 = v' + (v + v'') = (v' + v) + v'' = 0 + v'' = v''

Notazioni e ulteriori proprietà

  • La scrittura v - w è un'abbreviazione di v + (-w)
  • Vettore nullo: 0
  • L'opposto del vettore nullo -0 non è altro che 0
Proprietà importanti:
  1. v + w = z ⇒ v = z - w
  2. v + w = w ⇒ v = 0
  3. 0 · v = 0 = h · 0
  4. h · v = 0 ⇔ h = 0 oppure v = 0
  5. h(-v) = -(hv) = (-h)v
  6. (-1)v = -v
  7. Proprietà associativa generalizzata per n elementi
  8. Proprietà commutativa generalizzata per n elementi
  9. Proprietà distributiva generalizzata: (h₁ + h₂ + ... + hₙ)v = h₁v + h₂v + ... + hₙv

Esempi di Spazi Vettoriali

  1. Spazio vettoriale numerico di ordine n: (ℝⁿ, +, ·)
    • · : ℝ × ℝⁿ → ℝⁿ, (h, v = (v₁, ..., vₙ)) ↦ (hv₁, ..., hvₙ)
    • + : ℝⁿ × ℝⁿ → ℝⁿ, ((vᵢ), (wᵢ)) ↦ (vᵢ + wᵢ)
  2. Spazio vettoriale di una matrice di ordine m, n: (ℝₘ,ₙ, +, ·)
    • · : ℝ × ℝₘ,ₙ → ℝₘ,ₙ, (h, (aᵢⱼ)) ↦ (haᵢⱼ)
    • + : ℝₘ,ₙ × ℝₘ,ₙ → ℝₘ,ₙ, ((aᵢⱼ), (bᵢⱼ)) ↦ (aᵢⱼ + bᵢⱼ)
  3. Spazio vettoriale dei polinomi: (ℝ[x], +, ·) dove:
    • ℝ[x] = {a₀ + a₁x + a₂x² + ... + aₙxⁿ | aᵢ ∈ ℝ, n ∈ ℕ}
    • Principio di identità dei polinomi: a₀ + a₁x + ... + aₙxⁿ = b₀ + b₁x + ... + bₘxᵐ ⇔ n = m e a₀ = b₀, ..., aₙ = bₙ
  4. Spazio dei polinomi di grado ≤ n: (ℝₙ[x], +, ·)
  5. Spazio vettoriale dei vettori geometrici applicati in un punto: (S₀, +, ·)
  6. Spazio vettoriale dei vettori geometrici liberi: (V, +, ·)

Combinazione Lineare e Proporzionalità

Combinazione lineare: Presi h₁, ..., hₙ ∈ ℝ, v₁, ..., vₙ, v ∈ V, se è possibile scrivere v = h₁v₁ + ... + hₙvₙ allora v è detto combinazione lineare dei vettori vᵢ mediante gli scalari hᵢ.

Proporzionalità: Siano v, w ∈ V. Si dicono proporzionali se ∃h ≠ 0 ∈ ℝ tale che v = hw.
La proporzionalità è una relazione d'equivalenza essendo:
  • Riflessiva: v = 1 · v
  • Simmetrica: v = hw con h ≠ 0 ⇒ w = h⁻¹v
  • Transitiva: se h, k ≠ 0, v = hw e w = ku allora v = (hk)u

Sottospazi Vettoriali

Sia (V, +, ·) spazio vettoriale e sia H ⊆ V diverso dal vuoto. H è detto stabile (o chiuso) rispetto all'addizione se ∀v, w ∈ H ⇒ v + w ∈ H. H è detto stabile (o chiuso) rispetto alla moltiplicazione per uno scalare se ∀h ∈ ℝ, ∀v ∈ H ⇒ hv ∈ H.

Se H è chiuso rispetto a somma e prodotto allora è detto sottospazio vettoriale. Si indica con H ≤ V.
Esempi di sottospazi vettoriali:
  • Ogni spazio vettoriale V ha sempre i sottospazi banali: {0} e V
  • In ℝ²: H = {(x, y) | x = y} è un sottospazio
  • In ℝ⁴: H = {(a b; 0 a) | a, b ∈ ℝ} è un sottospazio
Controesempi:
  • H = {3x + 1, 0} ≰ ℝ[x] perché (3x + 1) + (3x + 1) = 6x + 2 ∉ H
  • H = {(x, y, 0, 0) ∈ ℝ⁴ | x = y²} ≰ ℝ⁴ perché (1,1,0,0) + (1,1,0,0) = (2,2,0,0) ∉ H

Teorema sui sottospazi

Se H ≤ V allora (H, +|ₕ, ·|ₕ) è spazio vettoriale.
Dimostrazione:
Tutte le proprietà che non sono esistenziali sono automaticamente soddisfatte per la stabilità dell'operazione. Ha lo stesso elemento neutro di V e contiene anche gli opposti (v ∈ H ≠ ∅ ⇒ (-1)v = -v ∈ H).

Proprietà e definizioni sui sottospazi (sottospazio generato)

  1. W ≤ V è stabile rispetto alla somma di n oggetti: Siano w₁, ..., wₙ ∈ W, si ha w₁ + ... + wₙ ∈ W. Qualunque combinazione lineare: h₁w₁ + ... + hₙwₙ ∈ W
  2. L'intersezione di sottospazi è un sottospazio: Sia L una famiglia di sottospazi di V, l'intersezione ⋂ᴸ∈ℒ L è un sottospazio
Sottospazio generato: Siano H, K ≤ V. Il più piccolo sottospazio che contiene sia H che K è: ⟨H, K⟩ = ⟨H ∪ K⟩ = H + K = {h + k | h ∈ H, k ∈ K}

Generalizzazione: ⟨H₁, ..., Hₙ⟩ = H₁ + ... + Hₙ = {h₁ + ... + hₙ | hᵢ ∈ Hᵢ}

Osservazione importante: Se ho v₁, ..., vₙ ∈ V, il sottospazio generato ⟨v₁, ..., vₙ⟩ = {h₁v₁ + ... + hₙvₙ | hᵢ ∈ ℝ} è l'insieme di tutte le possibili combinazioni lineari dei vettori v₁, ..., vₙ.

Somma diretta di sottospazi vettoriali

La somma di due sottospazi H₁, H₂ ≤ V si dice somma diretta se la loro intersezione è la più piccola possibile, ovvero H₁ ∩ H₂ = {0}. Si denota con H₁ ⊕ H₂.

H₁ e H₂ sono detti supplementari se V = H₁ + H₂, mentre sono complementari se V = H₁ ⊕ H₂.

Somma diretta di più sottospazi: H₁, ..., Hₙ sono in somma diretta se Hᵢ ∩ ⟨Hⱼ | j ≠ i⟩ = {0} per ogni i.
Esempi:
  • In ℝ²: H₁ = {(x, 0) | x ∈ ℝ}, H₂ = {(0, x) | x ∈ ℝ} ⇒ ℝ² = H₁ ⊕ H₂
  • H₁ = {(x, 0) | x ∈ ℝ}, H₂ = {(0, x) | x ∈ ℝ}, H₃ = {(x, x) | x ∈ ℝ} non sono in somma diretta

Dipendenza e Indipendenza Lineare

Siano v₁, v₂, ..., vₙ ∈ V. Essi sono detti linearmente dipendenti (o legati) se e solo se esistono n scalari non tutti nulli tali che h₁v₁ + ... + hₙvₙ = 0.

Se tali scalari non esistono allora i vettori sono detti linearmente indipendenti (o liberi), il che implica che l'unica soluzione è h₁ = h₂ = ... = hₙ = 0.
Esempi:
  • (1,0), (0,1) ∈ ℝ² sono linearmente indipendenti
  • (0,0), (1,0) sono linearmente dipendenti (l'elemento neutro implica la dipendenza)
  • (1,0), (2,0) sono linearmente dipendenti (due elementi proporzionali)

Dipendenza di un vettore da altri

Un vettore v dipende da v₁, v₂, ..., vₙ se v è combinazione lineare dei vᵢ, ovvero se ∃h₁, ..., hₙ ∈ ℝ : v = h₁v₁ + ... + hₙvₙ.

Proprietà:
  • 0 dipende sempre da qualunque sistema v₁, ..., vₙ
  • vᵢ dipende sempre da v₁, ..., vₙ
  • Proprietà transitiva: Se v dipende da v₁, ..., vₙ e ciascun vᵢ dipende da w₁, ..., wₘ, allora v dipende da w₁, ..., wₘ
  • Se v, w dipendono da v₁, ..., vₙ allora v + w dipende da v₁, ..., vₙ
  • v dipende da v₁, ..., vₙ ⇔ v ∈ ⟨v₁, ..., vₙ⟩

Relazione tra dipendenza e indipendenza lineare

Sia v₁, ..., vₙ ∈ V (con n ≥ 2). Allora:
  1. v₁, ..., vₙ sono linearmente dipendenti ⇔ ∃i : vᵢ dipende dai rimanenti
  2. v₁, ..., vₙ sono indipendenti ⇔ nessun vᵢ dipende dai rimanenti
Dimostrazione della relazione (1):
⇒: Per ipotesi esistono h₁, ..., hₙ ∈ ℝ non tutti nulli tali che h₁v₁ + ... + hₙvₙ = 0. Supponiamo h₁ ≠ 0, allora v₁ = (-h₁⁻¹h₂)v₂ + ... + (-h₁⁻¹hₙ)vₙ.
⇐: Supponiamo che v₁ dipenda dai rimanenti, quindi v₁ = h₂v₂ + ... + hₙvₙ. Allora 1·v₁ - h₂v₂ - ... - hₙvₙ = 0, quindi sono linearmente dipendenti.

Proposizioni sui sistemi indipendenti

  1. Siano v₁, ..., vₙ vettori linearmente indipendenti. Allora ogni vettore v che dipende da v₁, ..., vₙ vi dipende in maniera univoca.
  2. Siano H, K ≤ V non triviali, H ∩ K = {0}. Allora i vettori h ∈ H e k ∈ K (entrambi ≠ 0) sono indipendenti.
  3. Siano H₁, ..., Hₙ sottospazi in somma diretta, 0 ≠ vᵢ ∈ Hᵢ. Il sistema v₁, ..., vₙ è indipendente.
  4. Supponiamo di avere v₁, ..., vₜ indipendenti e v ∈ V che non dipende dai vᵢ. Allora v₁, ..., vₜ, v sono indipendenti.

Proposizioni sulla somma diretta

  1. Siano H₁ e H₂ sottospazi di V in somma diretta. Sia v ∈ H₁ ⊕ H₂. Allora ∃! v₁ ∈ H₁, v₂ ∈ H₂ : v = v₁ + v₂
  2. H₁, ..., Hₙ sono in somma diretta ⇔ ∀v ∈ H₁ + ... + Hₙ, ∃! hᵢ ∈ Hᵢ : v = h₁ + ... + hₙ

Vettori finitamente generabili

Uno spazio vettoriale V è detto finitamente generabile se esistono v₁, ..., vₙ : V = ⟨v₁, ..., vₙ⟩, dove ⟨v₁, ..., vₙ⟩ è detto sistema di generatori.

Definizioni associate:
  • Un sistema di generatori indipendente è detto base
  • Un riferimento è una base ordinata: e₁, ..., eₙ base ⇒ (e₁, ..., eₙ) riferimento
Esempi:
  • ℝⁿ[x] è finitamente generabile (generato da 1, x, x², ..., xⁿ)
  • ℝ[x] non è finitamente generabile
  • ℝ² ha base {(1,0), (0,1)}

Teorema di esistenza delle basi

Sia V uno spazio vettoriale non nullo. Se è finitamente generabile allora V ammette basi.
Dimostrazione:
Sia v₁, ..., vₙ un sistema di generatori. Se è indipendente, è già una base. Se è dipendente, esiste vᵢ che dipende dai rimanenti: lo si rimuove. Il nuovo sistema genera ancora V. Si itera fino ad ottenere un sistema indipendente.
Osservazione importante: Da ogni sistema di generatori si può estrarre una base.

Dimensione di uno spazio vettoriale

Lemma di Steinitz: Se ho m vettori linearmente indipendenti contenuti in un sottospazio generato da n vettori, allora m ≤ n.

Corollario: Sia V ≠ {0} spazio vettoriale. Tutte le basi hanno lo stesso ordine detto dimensione di V.

dim(V) è la cardinalità di una base. Se V = {0} ⇒ dim(V) = 0. Se V non è finitamente generabile ⇒ dim(V) = ∞.

Proposizioni della dimensione

Per V ≠ {0} finitamente generabile, dim V = n:
  1. v₁, ..., vₘ indipendenti ⇒ m ≤ n
  2. v₁, ..., vₙ indipendenti ⇒ v₁, ..., vₙ formano una base
  3. v₁, ..., vₙ sistema di generatori ⇒ v₁, ..., vₙ formano una base
  4. v₁, ..., vₘ indipendenti ⇒ ∃wₘ₊₁, ..., wₙ : v₁, ..., vₘ, wₘ₊₁, ..., wₙ formano una base (ogni sistema indipendente si può completare in una base)

Proposizione sui sottospazi

Sia W ≤ Vₙ. Allora:
  1. W è finitamente generabile e dim W ≤ n
  2. dim W = n ⇔ W = Vₙ
Corollari:
  • W ≤ Z ⇒ dim W ≤ dim Z
  • W ≤ Z e dim W = dim Z ⇒ W = Z

Proposizione sulla somma diretta

Siano H₁, ..., Hₙ ≤ V sottospazi in somma diretta. Allora: dim(H₁ ⊕ ... ⊕ Hₙ) = dim H₁ + ... + dim Hₙ

Relazione di Grassmann

Se H, K ≤ Vₙ sono sottospazi, allora: dim H + dim K = dim(H ∩ K) + dim(H + K)
Dimostrazione:
1. Si prende una base e₁, ..., eₜ di H ∩ K
2. La si completa a base e₁, ..., eₜ, h₁, ..., hᵣ₋ₜ di H e a base e₁, ..., eₜ, k₁, ..., kₛ₋ₜ di K
3. Si dimostra che e₁, ..., eₜ, h₁, ..., hᵣ₋ₜ, k₁, ..., kₛ₋ₜ è una base di H + K
4. Si conta: dim(H + K) = r + s - t = dim H + dim K - dim(H ∩ K)

Componenti di un vettore rispetto a un riferimento

Sia Vₙ uno spazio vettoriale di dimensione n e R = (e₁, ..., eₙ) un riferimento (base ordinata). Allora: ∀v ∈ Vₙ ∃! h₁, ..., hₙ ∈ ℝ : v = h₁e₁ + ... + hₙeₙ

La n-upla (h₁, ..., hₙ) si chiama n-upla dei componenti nel riferimento R.

Base canonica

La base canonica di ℝⁿ è: e₁ = (1,0,...,0), e₂ = (0,1,...,0), ..., eₙ = (0,0,...,1)

Altri esempi di basi canoniche:
  • Base canonica di ℝₙ[x]: 1, x, x², ..., xⁿ
  • Base canonica di ℝₘ,ₙ: matrici con un 1 in una posizione e 0 altrove

Metodi per estrarre una base da un sottospazio generato

Metodo 1: Eliminazione dei vettori proporzionali o dipendenti
Metodo 2: Riduzione a gradini
  • Si posizionano i vettori in una matrice
  • Si trasforma in matrice a gradini equivalente
  • Il numero di pivot = dimensione del sottospazio
  • Le righe non nulle formano una base
Esempio: H = ⟨(1,0,1,2), (-1,2,3,0), (3,-2,-1,4), (-2,4,6,0)⟩
Si nota la proporzionalità tra il secondo e quarto vettore, si eliminano i dipendenti, si ottiene dim H = 2.
